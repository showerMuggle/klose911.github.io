Kubernetes集群内部存在三类IP，分别是：

  • Node IP： 宿主机 的IP地址
  • Pod IP：使用 网络插件 创建的IP（如flannel），使 跨主机的Pod 可以互通
  • Cluster IP： 虚拟 IP，通过 iptables规则 访问服务
  在安装node节点的时候，节点上的进程是按照flannel -> docker -> kubelet -> kube-proxy的顺序启动的
